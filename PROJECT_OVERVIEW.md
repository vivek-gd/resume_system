# 简历管理系统项目介绍文档

## 1. 项目概述

简历管理系统是一个功能完整的个人简历在线管理平台，旨在帮助用户便捷地创建、编辑、管理和分享个人简历。系统支持PDF/Word简历上传解析、可视化编辑、版本历史管理、PDF导出等核心功能，为用户提供一站式简历管理解决方案。

### 主要功能亮点：
- ✅ PDF/Word简历智能解析
- ✅ 可视化富文本编辑
- ✅ 版本历史记录与回滚
- ✅ 版本差异对比
- ✅ 简历PDF导出
- ✅ 个人简历分享链接
- ✅ AI智能简历优化建议
- ✅ 响应式设计，支持多设备访问

## 4. 核心功能模块

### 4.1 用户认证模块

**功能描述**：负责用户注册、登录和权限控制，确保只有授权用户能够访问编辑和管理功能。

**关键功能**：
- 用户注册：创建新用户账号，生成唯一标识符
- 用户登录：验证用户凭据，建立会话
- 权限控制：通过`@login_required`装饰器保护需要登录的路由
- 密码安全：使用Werkzeug进行密码哈希存储

**核心代码**：
- 注册路由：`/register` (POST)
- 登录路由：`/login` (GET/POST)
- 退出路由：`/logout` (GET)

### 4.2 简历管理模块

**功能描述**：系统的核心模块，负责简历的创建、编辑、展示和管理。

**关键功能**：
- 简历创建：为新用户自动创建默认简历
- 简历编辑：可视化富文本编辑界面
- 简历展示：美观的简历展示页面
- 简历分享：生成个人简历唯一分享链接
- 简历公开/私有设置：控制简历的可见性

**核心代码**：
- 简历展示路由：`/` (GET)、`/profile/<unique_id>` (GET)
- 简历编辑路由：`/edit` (GET/POST)
- 用户仪表板：`/dashboard` (GET)

### 4.3 文件处理模块

**功能描述**：处理文件上传、解析和存储，支持PDF和Word简历的智能解析。

**关键功能**：
- 文件上传：支持PDF和Word文档上传
- 简历解析：自动提取文档中的姓名、教育、工作经历等信息
- 头像上传：支持用户头像上传和预览
- 文件存储：安全存储上传的文件

**核心代码**：
- 文件上传解析：`/upload_resume` (POST)
- 编辑页文件处理：`/edit` (POST)

### 4.4 版本管理模块

**功能描述**：记录简历的修改历史，支持版本回滚和差异对比。

**关键功能**：
- 自动版本记录：每次修改自动创建版本记录
- 版本历史查看：展示所有历史版本
- 版本回滚：恢复到指定历史版本
- 版本差异对比：可视化对比不同版本的差异

**核心代码**：
- 版本历史路由：`/history` (GET)
- 版本回滚路由：`/rollback/<history_id>` (GET)
- 版本差异对比：`/diff/<history_id>` (GET)

### 4.5 AI智能建议模块

**功能描述**：基于用户简历内容，提供智能优化建议，帮助用户提升简历质量。

**关键功能**：
- 内容分析：分析简历各部分内容的完整性和质量
- 个性化建议：针对不同字段提供具体优化建议
- 实时反馈：在编辑界面实时显示优化建议

**核心代码**：
- AI建议生成函数：`generate_ai_suggestions(resume)`
- 建议展示：集成到`/edit`路由中

### 4.6 导出与分享模块

**功能描述**：支持简历PDF导出和分享功能，方便用户投递简历。

**关键功能**：
- PDF导出：将简历导出为PDF格式
- 历史版本导出：支持导出指定历史版本
- 简历分享：生成唯一分享链接
- 公开设置：控制简历的公开/私有状态

**核心代码**：
- PDF导出路由：`/export_pdf/<int:history_id>` (GET)
- 当前版本导出：`/export_current_pdf` (GET)
- 分享链接：`/profile/<unique_id>` (GET)

### 4.7 系统维护模块

**功能描述**：负责系统的自动维护和清理，确保系统稳定运行。

**关键功能**：
- 定时清理：自动清理过期的版本记录
- 数据库初始化：首次运行自动创建数据库结构
- 错误处理：优雅处理各种异常情况

**核心代码**：
- 定时任务：`clean_old_histories()`
- 数据库初始化：`init_db()`

## 7. 系统工作流程

### 7.1 简历创建与编辑流程

1. **用户注册/登录**
   - 用户访问系统，注册新账号或登录现有账号
   - 系统验证用户凭据，建立会话

2. **简历初始化**
   - 首次登录时，系统自动为用户创建默认简历
   - 用户可以选择上传现有简历或手动编辑

3. **简历编辑**
   - 用户进入编辑页面，修改简历各部分内容
   - 系统实时提供AI优化建议
   - 用户可以上传PDF/Word简历进行自动解析

4. **保存与版本管理**
   - 用户点击保存按钮
   - 系统自动记录当前版本
   - 更新简历内容并保存到数据库

### 7.2 简历分享与导出流程

1. **简历分享**
   - 用户在仪表板获取个人简历唯一链接
   - 系统生成基于用户unique_id的分享URL
   - 他人通过链接访问用户的公开简历

2. **PDF导出**
   - 用户在编辑页或历史页选择导出PDF
   - 系统将简历HTML转换为PDF格式
   - 浏览器下载生成的PDF文件

### 7.3 版本管理流程

1. **版本记录**
   - 每次修改自动创建版本历史记录
   - 记录修改时间、操作人和修改内容

2. **版本查看**
   - 用户进入版本历史页面
   - 系统展示所有历史版本列表

3. **版本回滚**
   - 用户选择要恢复的历史版本
   - 系统记录当前版本，然后恢复到选定版本
   - 用户确认恢复结果

4. **版本对比**
   - 用户选择两个版本进行对比
   - 系统高亮显示不同之处

## 9. 部署与配置

### 9.1 环境要求

- Python 3.11+
- pip包管理器
- wkhtmltopdf (PDF导出必需)
- 操作系统：Windows/Linux/MacOS

### 9.2 快速开始

1. **安装依赖**
   ```bash
   pip install -r requirements.txt
   ```

2. **安装wkhtmltopdf**
   - Windows: 从官网下载安装
   - Linux: `sudo apt-get install wkhtmltopdf`
   - MacOS: `brew install wkhtmltopdf`

3. **配置系统**
   - 修改 `app.py` 中的密码设置
   - 调整其他配置参数（如上传限制等）

4. **启动服务**
   ```bash
   python app.py
   ```

### 9.3 生产环境部署

**使用Gunicorn部署**：
```bash
pip install gunicorn
gunicorn -w 4 -b 0.0.0.0:5000 app:app
```

**使用Nginx反向代理**：
```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /static {
        alias /path/to/resume-system/static;
    }
}
```

**使用Docker部署**：
```dockerfile
FROM python:3.9-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt
RUN apt-get update && apt-get install -y wkhtmltopdf
COPY . .
CMD ["gunicorn", "-w", "4", "-b", "0.0.0.0:5000", "app:app"]
```
